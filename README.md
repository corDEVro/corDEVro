### 🚀 Proyectos Destacados

*   **[ApplyFlow](https://github.com/corDEVro/applyflow)** - API REST empresarial para la gestión y automatización del flujo de ofertas laborales. Diseñada bajo arquitectura limpia para centralizar el ciclo de vida de procesos de selección (Aplicado, Entrevista, Oferta, Rechazado).
    * **Backend Stack:** `Java 21`, `Spring Boot 3`, `Spring Security & JWT`, `Spring Data JPA`, `PostgreSQL`.
    * **Highlights Técnicos:**
        * **Seguridad Robustecida:** Implementación de autenticación stateless mediante tokens JWT y control de acceso basado en roles (RBAC) para proteger los datos privados de cada usuario.
        * **Automatización de Reglas de Negocio:** Lógica en el backend para el cálculo inteligente de alertas de seguimiento (Smart Follow-Up) cuando una postulación supera los días límite sin interacción.
        * **Persistencia y Optimización:** Modelo relacional en PostgreSQL gestionado eficientemente con Spring Data JPA y validaciones estrictas a nivel de API (`@Valid`) con manejo global de excepciones.
*   **[EDAR-Twin](https://github.com/corDEVro/edar-twin)** `[En Desarrollo / WIP]` - Plataforma de Gemelo Digital para la simulación, monitorización y optimización de Estaciones Depuradora de Aguas Residuales (EDAR). Diseñado para procesar flujos de datos complejos y estados de plantas industriales en tiempo real.
    * **Backend Stack:** `Java 21`, `Spring Boot 3`, `Spring Data JPA`, `PostgreSQL`, `Docker`.
    * **Highlights Técnicos:**
        * **Modelado de Sistemas Complejos:** Abstracción y arquitectura en el backend de los procesos físicos y químicos de una planta de tratamiento (reactores, decantadores) mediante POJOs estructurados y patrones de diseño.
        * **Procesamiento de Datos e Ingesta:** Diseñado con una arquitectura extensible preparada para la integración de telemetría IoT (simulada o vía brokers de mensajería) para actualizar las métricas del gemelo digital de forma reactiva.
        * **Estrategia de Persistencia:** Gestión de históricos de datos y estados de la planta optimizando las consultas relacionales en PostgreSQL para evitar cuellos de botella en la analítica de rendimiento.
